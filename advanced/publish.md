# 发布插件

如何打包、发布和分享你的 UniQQ 插件。

---

## 1. 编译 Release 版本

```bash
dotnet build -c Release
```
输出目录：`bin/Release/net10.0/`
---

## 2. 发布文件清单

插件发布需要以下**最少文件**：

```
你的插件文件夹/
├── 你的插件.dll          # 主程序集（含嵌入的依赖）
└── plugin.json            # 插件信息
```

## 3. 打包规范

### 文件夹结构

建议按照以下结构打包：
!!! 编译打包的插件本体dll文件命名必须与“plugin.json”的"entry"保持一致

!!!  (不要放在目录内通过压缩目录的方式打包！)

```
插件名_版本号/
├── plugin.json
└── 插件名.dll
```

然后打包为 RAR/ZIP 文件，压缩完成后将压缩包后缀改为"uniqq"

（不改压缩包后缀会导致框架不识别压缩包与插件）
## 4. 加载插件(两种)
1. UniQQ -> 插件 -> 安装新插件，选择你的插件
2. 将压缩后的uniqq后缀的压缩包放至插件根据目录中的“Plugins”目录即可，框架会自动识别插件资源
